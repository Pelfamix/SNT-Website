Contexte


Log4Shell (CVE-2021-44228) est une vulnérabilité dans Log4j, une libraire de logging Java développée par Apache, permettant une exécution de code arbitraire, c'est à dire une action à faire faire à une machine sans que le propriétaire soit d'accord.
La vulnérabilité, qui existe sans être remarquée depuis 2013, a été divulguée en privé à Apache par un chercheur de l'équipe de sécurité d'Alibaba Cloud le 24 novembre 2021. Ils ont développé un patch pour pouvoir la solutionner, et a été révélée publiquement le 9 décembre 2021, lorsqu'Apache a sorti son patch 2.15.
Cependant, après quelques tests, il y avait encore des vulnérabilité qui subsistaient. Apache a sorti le patch 2.16, qui permet de régler, à priori, la quasi-totalité des vulnérabilités qui ont été découvertes à ce jour.


Impact


Le nombre de systèmes et le nombre d'applications qui sont impactés par cette vulnérabilité est très importante. Log4j est utilisée par énormément d'applications, et la quasi-totalité des versions de cette librairie est concernée (toutes les versions 2.X sauf 2.12.2, 2.15 et 2.16). Ces applications peuvent être par exemple des programmes privés, mais peuvent aussi être des programmes commerciaux publics, notamment Minecraft: Java Edition, Steam, Amazon Web Services, etc.
